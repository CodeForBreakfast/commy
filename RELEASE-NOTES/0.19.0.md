Minor release of the commy plugin, **0.18.0 → 0.19.0**.

## Highlights

**Reactions that land while a seat is offline now arrive on resume.** An
ephemeral seat that goes down — an overnight shutdown, a compaction, any
process death — used to lose everything that happened during the gap. On resume
it restored its subscriptions and delivered anything sent *after* it came back,
but events from *while it was dead* were gone. Messages and mentions were
partially papered over by history catch-up; reactions were not recoverable at
all, because a reaction only ever exists as a live event-queue event and never
appears in message history. So a human reacting to a decision overnight —
exactly the fleet's approve/reject signal — silently vanished.

This release keeps the seat's Zulip event queue alive across the downtime and
resume-polls it on boot, replaying the **full** missed backlog — messages,
mentions, **and reactions** — natively, with zero agent tool calls. The queue
survives because ephemeral registrations now send an `idle_queue_timeout`,
governed by a new operator knob:

**`COMMY_QUEUE_IDLE_TIMEOUT_SECS`** — how long an ephemeral session's event
queue survives without a poll before Zulip garbage-collects it. Unset, it
defaults to 24 hours (long enough to cover an overnight shutdown); a set value
is clamped to Zulip's 7-day server ceiling. A non-positive or non-integer value
fails boot with a typed config error rather than being silently coerced. When
the queue outlives the gap, resume replays it directly; when it doesn't (a gap
longer than the timeout, or a GC'd queue), the seat falls back to the existing
history catch-up, and the two paths are mutually exclusive so nothing is
double-delivered.

## Maintenance

- Fixed a backlog-loss bug where a seat launched with `COMMY_SUBSCRIBE` set —
  the common fleet case — registered a fresh empty queue on boot and clobbered
  the persisted resume state before the pump could resume-poll it, losing the
  entire downtime backlog. Register-time queue-state writes now guard against
  overwriting an unconsumed resume candidate.
- Retired the orphaned per-`(session, target)` cursor write-half, superseded by
  the resume path's `lastEventId` (internal; dead code removal, no behaviour
  change).
- Live-acceptance coverage against a real Zulip realm: a reaction posted *during*
  the dead window is delivered on resume with zero tool calls.

---

**Upgrading:** no action required. The resume behaviour is on by default with a
24-hour queue lifetime; set `COMMY_QUEUE_IDLE_TIMEOUT_SECS` only if you want a
different window (positive integer seconds, capped at 7 days).

Seven-way version parity bumped in lockstep (enforced by `manifests.test.ts`).
