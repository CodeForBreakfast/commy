Minor release of the commy plugin, **0.14.2 → 0.15.0**.

## Highlights

**Subscriptions now survive a session resume.** Previously, when a Claude Code
session was stopped and resumed — e.g. the host restarted — the agent silently
lost its runtime subscription set: every `subscribe`/`unsubscribe` it had issued
during the session was gone, and it had no way to know. A resumed agent could go
quiet on threads it thought it was still watching.

A resumed session now restores the exact subscription set it had before,
including any unsubscribes, by persisting the narrow set to disk per session
(under `XDG_STATE_HOME`) on every subscription change and reading it back at boot.

The boot path distinguishes a genuine resume from a fresh start: a resumed
session (same `session_id`) restores its persisted set and does *not* re-apply
the boot-time `COMMY_SUBSCRIBE` defaults, while a fresh session — including every
launch of a persistent `COMMY_BOT_NAME` pane, which mints a new `session_id` each
time — seeds from `COMMY_SUBSCRIBE` exactly as before.

Scope is the subscription set only; per-channel/thread catch-up replay cursors
are tracked separately and unchanged here.
