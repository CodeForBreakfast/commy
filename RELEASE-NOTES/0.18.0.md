Minor release of the commy plugin, **0.17.0 → 0.18.0**.

## Highlights

**`download_file` can now land its result somewhere the caller can actually
read it.** Until now `download_file` always wrote the fetched attachment into a
fresh directory under `$TMPDIR`. That works when you have an unrestricted shell,
but for the allowlisted, headless agents commy is built for it was a dead end:
the file downloaded fine, yet the Read tool — confined to the agent's configured
working directories — couldn't open it. This release honours a new operator-set
`COMMY_DOWNLOAD_DIR`. When set, each download's per-call temp directory is
created under it, so the file lands inside a directory the agent is allowed to
Read; when unset, the `$TMPDIR` behaviour is unchanged. commy's MCP server is
per-agent, so the launch environment is the right per-caller root — the caller
never has to name a destination. The base directory is validated once at boot
(a `COMMY_DOWNLOAD_DIR` that isn't an existing directory fails boot with a typed
config error, rather than surfacing on the first download).
